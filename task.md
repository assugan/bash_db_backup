# Задача Резервное копирование СУБД
## Требуется написать на BASH скрипт резервного копирования СУБД MySQL или PostgreSQL (на выбор), работающий согласно алгоритму:
- Определяется список баз данных.
- Для каждой базы создается дамп штатными средствами.
- После создания все дампы помещаются в общий gzip-архив.
- Архив тестируется на целостность.
- Архив переносится в каталог /backups (считаем, что это отдельный диск под бэкапы).

## Особое внимание уделить следующим аспектам:
### Логгирование:
- Весь процесс работы скрипта и отработки бэкапа должен сопровождаться записью сообщений в лог-файл.
- Сообщения должны быть лаконичными, составлены своими словами и не содержать тонну нечитаемого технического мусора.
- Успешный либо неуспешный результат каждого действия скрипта должен быть отражен в лог-файле.
### Обработка нештатных ситуаций:
- Сообщение о возникновении нештатной ситуации или ошибки должно появиться в лог-файле.
- После возникновения ошибки не выполнять дальнейшие действия, кроме логирования. Например, не надо создавать архив, если перед этим не смогли корректно выполниться дампы.
- Примеры нештатных ситуаций:
    - неправильный пароль от базы,
    - создание дампа завершилось ошибкой,
    - закончилось свободное место на одном из дисков,
    - тестирование архива завершилось неудачей и т.д.
### Состояние после работы скрипта:
- При любом сценарии отработки, по окончании работы скрипта на диске не должно оставаться никакого мусора:
    дампов  БД, временных файлов и т.д.
- В каталоге /backups в итоге должны лежать только исправные бэкапы.
